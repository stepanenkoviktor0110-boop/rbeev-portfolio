# LevelUPMe

Портфолио-сайт фотографа на `Next.js` с публичной частью и админ-панелью.

## Что сейчас реализовано

- Главная страница: `Hero` → `Календарь съемок` → `Галерея` → `Обо мне` → `Как всё устроено` → `Контакты`
- Адаптивная верхняя навигация:
  - бренд «Ваш фотограф — Рамазан Беев» автоматически скрывается, если не помещается
  - при нехватке места приоритет у ссылок разделов (бренд уходит, разделы остаются видимыми)
  - на мобильных ссылки выровнены по центру
  - мобильная панель навигации сделана крупнее (высота и внутренние отступы)
  - шрифт ссылок и расстояние между пунктами увеличены для лучшей читаемости
- Иконки сайта (favicon/PWA):
  - `РБ` на тёмном фоне в фирменном цвете
  - отдельные файлы для вкладки, адресной строки и мобильных устройств
  - `favicon.ico` дополнительно оптимизирован под маленькие размеры (16/24/32/48/64)
- Лента доступности дат (10 дней от завтрашнего дня, автоматически):
  - если период попадает на два месяца, подпись показывается как диапазон (например, `Март-Апрель`)
  - карточки дат в ленте сделаны шире для лучшей читаемости
  - занятые даты выделены более насыщенным красным акцентом
- Hero-слайдер:
  - на каждом HTTP-запросе выбирает 5 случайных фото из отмеченных `showInSlideshow`
  - безопасный fallback на SSR-набор фото, если запрос случайной пятерки недоступен
  - плавный crossfade-переход между кадрами (без резкой смены)
  - облегченный «киношный» оверлей без тяжелых runtime-фильтров
  - усиленный контраст заголовка и подзаголовка
- Клик по свободной дате:
  - автоподстановка в поле «Желаемая дата фотосессии»
  - плавный скролл к форме
  - подсветка поля даты
- Галерея с фильтрами и лайтбоксом
- В публичной галерее не показываются категории без фото
- Лайтбокс без лишнего UI:
  - без заголовка/счетчика/крестика
  - закрытие кликом вне фото
  - нециклическая навигация
  - клик по фото листает вперед; на последнем фото закрывает лайтбокс
  - клавиши `←`/`→`, свайп на мобильных
- Блок «Обо мне» без ручного UI-слайдера (медленный автокроссфейд)
- Блок «Как всё устроено» в 3 колонки + отдельная строка стоимости
- Форма заявок:
  - inline-валидация
  - обновленный текст согласия
  - основная CTA-кнопка: `Договориться о съёмке`
  - thank-you экран после отправки
- Блок отзывов после формы заявки (данные управляются через админку)
- Админка: Галерея, Категории, Заявки, Отзывы, Настройки
- Оптимизации загрузки:
  - изображения галереи идут через оптимизацию `next/image` (без `unoptimized`)
  - лайтбокс также использует `next/image`
  - главная страница кэшируется с `revalidate = 60`

### Хранение и отображение фото

Фотографии хранятся на **Яндекс.Диске** вашего аккаунта. Браузеру изображения
никогда не отдаются напрямую с Яндекса — они проксируются через маршрут
`/api/photos/image?key=...`, который на каждый запрос получает свежую
подписанную ссылку и стримит содержимое через наш сервер.

### Два режима загрузки фото в админке

**1. Загрузка файлов (1–10 за раз)**
- Файлы загружаются на ваш Яндекс.Диск в папку `site-gallery`
- Прогресс считается по реально переданным байтам

**2. Импорт папки по ссылке (без копирования)**
- Вставьте публичную ссылку на папку Яндекс.Диска (Поделиться → Скопировать ссылку)
- Сервер читает список файлов без скачивания — папка остаётся на вашем диске нетронутой
- Автоматически создаётся категория с именем папки
- Все фото сразу показываются на витрине (`showInGallery = true`)
- Ссылки вида `/client/disk/...` не принимаются

### Управление занятыми датами

Вынесено в `Заявки → Календарь`. Без заявки занять дату нельзя.
Подробнее: [PROJECT_DOCS.md](PROJECT_DOCS.md).

## Быстрый старт

1. Установить зависимости:

```bash
npm install
```

2. Создать `.env`:

```powershell
Copy-Item .env.example .env
```

3. Заполнить переменные окружения:

| Переменная | Описание |
|---|---|
| `DATABASE_URL` | PostgreSQL connection string |
| `AUTH_SECRET` | Секрет сессии (мин. 16 символов) |
| `ADMIN_PASSWORD` | Пароль в открытом виде для первого входа |
| `NEXT_PUBLIC_SITE_URL` | Публичный URL сайта (без слеша в конце) |
| `YANDEX_DISK_TOKEN` | OAuth-токен вашего Яндекс.Диска |
| `YANDEX_DISK_BASE_DIR` | Базовая папка для прямых загрузок (например, `/site-gallery`) |

> `YANDEX_DISK_TOKEN` должен быть токеном именно вашего Яндекс-аккаунта.
> При импорте папки файлы не копируются — используется папка, которую вы указываете по ссылке.

4. Подготовить БД:

```bash
npx prisma generate
npx prisma db push
npx prisma db seed
```

Для существующей БД с данными:

```bash
npx prisma migrate deploy
```

5. Запустить проект:

```bash
npm run dev
```

- Сайт: `http://localhost:3000`
- Админка: `http://localhost:3000/admin`

## Сброс пароля админки

Пароль хранится в БД только в виде `bcrypt`-хэша.

1. Задайте новый `ADMIN_PASSWORD` в `.env`.
2. Очистите поле `Settings.adminPasswordHash` для записи `id = 1` (или удалите запись `Settings`).
3. При следующей авторизации приложение создаст новый хэш.

## Архитектурный ориентир

- Цель: любой контентный блок на публичной части управляется через админ-панель.
- Текущее исключение: блок «Как всё устроено» рендерит фиксированные тексты из кода.
  `Settings.workflowItems` сохраняется в БД, но публичным компонентом пока не используется.
