# LevelUPMe

Портфолио-сайт фотографа на `Next.js` с публичной частью и админ-панелью.

## Что сейчас реализовано

- Главная страница: `Hero` -> `Календарь съемок` -> `Галерея` -> `Обо мне` -> `Как всё устроено` -> `Контакты`
- Лента доступности дат (10 дней от завтрашнего дня, автоматически)
- Клик по свободной дате:
  - автоподстановка в поле «Желаемая дата фотосессии»
  - плавный скролл к форме
  - подсветка поля даты
- Галерея с фильтрами и лайтбоксом
- Лайтбокс без лишнего UI:
  - без заголовка/счетчика/крестика
  - закрытие кликом вне фото
  - нециклическая навигация
  - клик по фото листает вперед; на последнем фото закрывает лайтбокс
  - клавиши `←`/`→`, свайп на мобильных
- Блок «Обо мне» без ручного UI-слайдера (медленный автокроссфейд)
- Блок «Как всё устроено» в 3 колонки + отдельная строка стоимости
- Для блока «Как всё устроено» сейчас используется фиксированный текст карточек на витрине
- Пункт «Как всё устроено» добавлен в верхнее меню
- Форма заявок:
  - inline-валидация
  - обновленный текст согласия
  - thank-you экран после отправки
- Блок отзывов после формы заявки (данные управляются через админку)
- Админка: Галерея, Категории, Заявки, Настройки
- Админка: добавлен раздел `Отзывы` (create/edit/delete/publish/order)
- Ускорены ключевые действия в админке (меньше лишних перезагрузок списков)

## Важное изменение по занятым датам

Управление занятыми датами вынесено в отдельную сущность `BusyDate` и UI в разделе `Заявки`:

- вверху страницы `Админка -> Заявки` есть календарь на 10 ближайших дней
- вручную занять дату без заявки нельзя
- при клике на свободную дату открывается форма создания заявки на эту дату
- дату можно подтвердить из карточки заявки кнопкой «Подтвердить дату в календаре»
- такая же кнопка есть в режиме редактирования заявки

Хранение в `Settings.busyDates` больше не используется для календаря.

## Быстрый старт

1. Установить зависимости:

```bash
npm install
```

2. Создать `.env`:

```powershell
Copy-Item .env.example .env
```

3. Заполнить переменные окружения:

- `DATABASE_URL`
- `AUTH_SECRET` (минимум 16 символов)
- `ADMIN_PASSWORD` (минимум 8 символов)
- `NEXT_PUBLIC_SITE_URL`
- `CLOUDINARY_CLOUD_NAME`
- `CLOUDINARY_API_KEY`
- `CLOUDINARY_API_SECRET`

4. Подготовить БД:

```bash
npx prisma generate
npx prisma db push
npx prisma db seed
```

5. Запустить проект:

```bash
npm run dev
```

- Сайт: `http://localhost:3000`
- Админка: `http://localhost:3000/admin`

## Архитектурный ориентир проекта

- Цель: любой контентный блок на публичной части должен управляться через админ-панель.
- Текущее исключение: блок «Как всё устроено» на витрине рендерит фиксированные 3 карточки и строку стоимости из кода.
- Поле `workflowItems` в админке пока сохраняется в `Settings`, но не используется публичным компонентом.
